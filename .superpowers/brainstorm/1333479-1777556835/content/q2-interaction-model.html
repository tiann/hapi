Cậu tưởng tượng workflow giữa editor và session chat như thế nào?

Mối quan hệ giữa panel chat (phải) và editor (giữa)

A

1 session chat gắn với 1 project

Panel phải có 1 session chat duy nhất cho cả project. AI toàn quyền truy cập toàn bộ project. Mình mở file nào thì AI cũng biết và có thể edit. Giống antigravity — 1 cuộc hội thoại xuyên suốt cho project.

B

Multiple sessions, chọn session để chat

Panel phải có nhiều session chat (tabs hoặc list). Mỗi session độc lập — có thể dùng cho các task khác nhau. Khi chat session nào, context của AI giới hạn trong session đó. File vẫn mở chung ở editor.

C

Multiple sessions + tự động gắn file đang mở

Giống B, nhưng AI trong session tự động biết mình đang xem file nào ở editor. Mở tab mới → AI được context về file đó. Đóng tab → context tự thu hẹp. Tương tự cursor composer với open tabs.
