3 Approaches cho Editor Mode

Từ nhanh nhất đến tối ưu nhất

A

MVP thẳng tiến — Route mới + ghép component có sẵn

Thêm route /editor, layout 3 cột flexbox. Panel trái = DirectoryTree, giữa = CodeMirror viewer, phải = SessionList + SessionChat. Dùng nguyên component hiện tại, chỉ viết layout mới + editor viewer. Nhanh nhất.

Pros

  • ~5-6 ngày
  • Rủi ro thấp — không sửa code hiện tại
  • Dễ rollback nếu fail

Cons

  • Code chưa tối ưu — SessionChat hơi nặng cho panel phải
  • Không có tab system — mở từng file một
  • 2 page riêng biệt, chưa có cross-link mượt
B

MVP + Tab System + Compact Chat

Tất cả của A, thêm: (1) Tab system cho editor — mở nhiều file, chuyển tab, close tab. (2) Compact SessionChat — fork nhẹ bỏ voice/terminal/team panel. (3) Context menu "Add to Chat". Đầy đủ tính năng cốt lõi.

Pros

  • Trải nghiệm gần giống VS Code/Cursor
  • Tab system + context menu mạnh
  • Tách biệt rõ với agent mode

Cons

  • ~7-9 ngày
  • Fork SessionChat = duplicate code
  • File write API cần thêm (phase 2)
C

Full-featured — Refactor + Phase 2 ready

Tất cả của B +: (1) Refactor SessionChat thành composable (ChatCore, ComposerBar) — dùng chung cho cả 2 mode. (2) File write RPC sẵn sàng. (3) Monaco Editor thay vì CodeMirror viewer. Sẵn sàng cho phase 2 (full edit).

Pros

  • Kiến trúc sạch, bền vững
  • Ready cho phase 2
  • 1 source of truth cho chat component

Cons

  • ~10-12 ngày
  • Refactor SessionChat rủi ro break agent mode
  • Monaco Editor nặng (~5MB bundle)
